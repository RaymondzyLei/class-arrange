# 课程详情视图优化设计

## 目标

在不改变课程数据和选择逻辑的前提下，提升课程详情弹窗的信息密度、长文本可读性和展开交互；同时修复参考书字段首尾多余引号，并调整新手引导内容。

## 标题与课程简介

- 为 `BottomModal` 增加标题附加操作区域，使课程名、简介开关、选课操作和关闭按钮保持同一基线。
- “查看课程简介”紧邻课程标题右侧，使用无边框文字按钮。
- 按钮右侧复用现有 `ChevronIcon` 与 `select-chevron` 旋转样式；展开时箭头旋转，收起时复位。
- 简介内容继续显示在详情信息上方，但不再使用额外外框。
- 简介容器保持挂载，通过 CSS Grid 高度、透明度和间距过渡完成展开与收起动画。切换课程时默认恢复收起状态。
- 保留 `aria-expanded`、`aria-controls` 和简介区域语义，收起状态不向辅助技术暴露正文。

## 详情信息结构

采用已选定的 A“分组信息卡”方向，但基础字段全部保留在带边界的详情表格中，不使用胶囊标签或 Tag 展示普通信息。

详情表格包含：

- 课堂号 / 班次
- 开课单位
- 授课教师
- 学分 / 学时
- 课程类型
- 考核方式
- 评分制
- 授课语言
- 本研同堂（存在时）
- icourse 评分（存在时）

表格使用紧凑行高和稳定的标签列宽；教师等长内容占整行，避免当前宽弹窗中出现大块无效空白。

教材与参考资料放在表格下方的独立内容区：

- 参考书单独成组并允许自然换行。
- 教材与讲义分别显示，保留结构化教材信息。
- 普通字段不使用 Tag；现有“本研同堂”状态可以继续使用语义明确的状态 Tag。
- 桌面端保持紧凑分组，移动端按单列顺序展示，不丢失字段。

班次明细和时间地点表格保留现有数据与交互，只统一标题间距、表格圆角和长文本换行。

## 参考书引号修复

- 在课程资料格式化层清理参考书字段首尾成对的英文双引号。
- 先去除首尾空白，再仅在首字符和尾字符均为 `"` 时剥离这一对包裹引号。
- 不删除正文内部引号，不删除中文书名号，也不处理只有单侧引号的内容。

## 新手引导调整

- 删除当前第 10/11 步“调整排课倾向”（`customization-preferences`）。
- 保留“设置占位时间”，删除后其编号自动前移，引导总步骤数同步减少。
- 第 2 步继续介绍多种排课方案，但预览图片替换为用户提供的 39 门、97.5 学分、8 种方案截图。
- 图片作为静态受版本控制资源存放在 `src/assets/onboarding/arrangement-preview.png`。

## 测试与验证

- 组件测试验证简介按钮位于标题附加区域、无边框、带箭头并正确输出展开状态。
- 组件测试验证简介正文收起时不可见、展开时完整显示，并具备动画容器状态类。
- 资料格式化测试覆盖成对外引号、单侧引号和正文内部引号。
- 引导内容测试验证 `customization-preferences` 已移除且第 2 步仍引用静态 PNG。
- 使用内置浏览器分别检查桌面和移动宽度下的标题对齐、详情表格、展开动画及引导第 2 步截图。
