# 搜索、状态与引导界面修复设计

## 目标

修复排课状态文案重复、学期切换箭头闪烁和引导说明缺失；重新制作第 2/11 步的真实示例截图；移除搜索区容易误解的课程数量，并把教师查询开关移动到搜索框右侧。

## 界面行为

### 排课状态

- 状态栏只渲染一个主要文本节点，不再同时显示粗体标题和补充说明。
- 状态栏与排课方案列表合并在同一张卡片中，并用分隔线区分；搜索筛选仍保持独立卡片。
- 课程或偏好变化时显示：`课程或偏好已变更。`
- 最近一次计算成功时显示：`当前课表来自最近一次成功计算。`
- 未选择课程、计算中和错误状态仍保留各自必要的一句提示及原有操作按钮。

### 学期切换

- 学期按钮在加载期间继续显示同一个展开/收起箭头。
- 加载期间禁用按钮并设置忙碌状态，但不使用会替换图标的按钮 `loading` 外观。
- 菜单关闭后箭头按现有动画回到收起状态。

### 搜索区域

- 完整删除 `共 N 门` 及其 `resultCount` 数据链路，不再展示课程数或时间组数。
- 把现有 `查询任课老师` 复选框从下方筛选网格移动到搜索框右侧。
- 搜索行使用 `minmax(0, 1fr) max-content` 两列和 8px 间距；教师复选框保持单行，输入框允许收缩但不溢出。
- 下方六个筛选器保持三列两行，移动端沿用同一搜索行；输入框占据复选框以外的全部可用宽度。

课程数据本身不做删减。2026 秋季数据是 2484 个课堂、1381 个唯一课程号；此前显示的 1917 是按上课时间拆分后的选课时间组数量。

### 首次使用与第 2/11 步

- 在“选择使用偏好”标题下新增：`稍后可在“自定义”中修改设置`。
- 使用修改后的真实页面状态，准备能产生多种排课方案的示例选课。
- 从浏览器截取包含单行状态提示和多张排课方案卡片的 PNG，保存为受版本控制的引导资源。
- 第 2/11 步用该真实 PNG 替换当前代码生成的模拟排课面板，并更新说明，明确“我的方案”和多种“排课方案”的关系以及最多展示 8 种结果。
- 图片仅用于说明，不参与交互；在桌面和窄屏容器内等比缩放。

## 验证

- 状态组件回归测试验证两个指定文案且不再输出旧标题。
- 学期按钮回归测试验证加载期间箭头仍存在。
- 搜索栏回归测试验证数量信息消失、教师复选框位于搜索行且不再要求 `resultCount`。
- 引导回归测试验证新增说明和第 2/11 步引用真实图片。
- 浏览器检查搜索行宽度、学期切换过程和第 2/11 步图片效果。
- 最后运行相关测试与生产构建各一次。
