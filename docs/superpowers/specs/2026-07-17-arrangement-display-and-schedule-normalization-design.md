# 排课方案展示数量与时间组归并设计

## 目标

本次修改包含两个相互独立但共同影响排课结果展示的范围：

1. 允许用户在“自定义”设置中选择常规展示的排课方案数量，并能按需查看全部无冲突方案。
2. 修正课程时间表分段方式不同导致同一时间组未被合并的问题。

不改变首次进入时的偏好设置，不修改现有开关样式，也不改变排课方案的排序规则。

## 自定义设置

在“自定义 → 课表生成”分组新增“展示排课方案数量”下拉框，选项固定为 `2`、`4`、`8`、`12`、`16`，默认值为 `8`。

该设置不出现在首次进入时的偏好设置页面。设置变更后触发重新排课，常规方案列表按用户选择的数量保留和展示；设置需要与其他自定义设置一样持久化。读取旧版本设置时，缺少该字段应自动使用默认值 `8`。

## 无冲突方案展示

排课方案板块增加独立按钮“展示所有不冲突方案”。它不是设置项，也不改变用户保存的常规展示数量。

每次常规排课时，计算引擎仍完成现有的精确组合遍历，同时生成三类结果：

- 按现有排序规则保留用户设置数量的推荐方案；
- 按同一排序规则保留最多 `100` 个冲突数为零的方案；
- 统计全部冲突数为零的方案数量。

现有引擎本来就会遍历全部候选组合，因此统计无冲突方案总数只增加常数级判断和计数，不增加组合搜索的数量级。总组合数非常大时，计算耗时仍受现有穷举算法限制。

点击“展示所有不冲突方案”后，界面立即切换到最多 `100` 条的无冲突预览。此时提供“返回推荐方案”，用于恢复用户设置数量的常规方案列表，并保持当前常规方案的选择状态。

无冲突模式的底部信息规则如下：

- 总数不超过 `100`：显示“共 xx 种不冲突方案”。
- 总数超过 `100`：显示“共 xx 种不冲突方案，全部展示。”，其中“全部展示”可点击。
- 总数为 `0`：显示明确的无结果提示，并允许返回推荐方案。

点击“全部展示”后，使用专门的计算请求重新遍历组合并收集全部无冲突方案。加载期间禁用重复触发并显示加载状态；计算失败或被新设置变更取代时，保留原有最多 `100` 条预览和准确总数，不清空当前结果。成功后在同一方案列表中展示全部结果。

“加载全部”是用户明确触发的高开销操作。常规计算不长期保存所有无冲突方案，避免无条件增加内存和 Worker 传输成本。

## 计算接口与状态

计算引擎接收明确的结果策略：

- `recommended`：返回设置数量的推荐方案、最多 `100` 条无冲突预览及无冲突总数。
- `all-conflict-free`：只收集并返回全部零冲突方案及其总数。

Worker 协议返回结构化结果，而不是只有方案数组。应用层将“已提交的常规计算结果”和“按需加载全部的状态”分开保存，防止较晚完成的旧请求覆盖设置变更后的新结果。任何设置或课程选择变化都退出无冲突模式并以新输入重新计算。

方案选择继续使用稳定的方案 ID。返回推荐模式时恢复此前选中的推荐方案；如果无冲突模式中选择了其他方案，只在该模式和输入仍有效时保留。

## 时间组归并修复

时间组归并比较“实际占用时间”，而不是比较原始时间段数组如何切分。

生成时间指纹时，将每个日程展开为按周的原子占用单元。每个单元包含：

- 周次；
- 星期；
- 节次，或无节次数据时的精确起止时间；
- 校区。

教室继续按照现有规则不参与时间组指纹。展开后的单元去重、排序后生成规范指纹。因此，`1～9周 + 10～18周` 与 `1～18周` 在星期、节次和校区相同时得到相同指纹，能够归入同一时间组；如果中间缺周、星期不同、节次不同或校区不同，则仍属于不同时间组。

规范化只用于比较时间占用。课程原始教师、教室和分段日程继续保留，详情展示不虚构“某位老师对应某段时间”的映射，因为当前课程数据只提供合并后的教师字段，没有逐日程教师关系。

## 错误与边界处理

- 持久化设置值不在允许选项中时回退到 `8`。
- 无冲突总数以实际精确遍历结果为准，不使用估算值。
- “全部展示”请求过期、取消或失败时不覆盖已提交的常规结果。
- 全部结果可能很大；只有用户点击“全部展示”才承担相应内存和渲染开销。
- 日程缺少周次时保持现有语义，不能因规范化而与具有明确周次的日程误合并。

## 测试与验收

自动化测试至少覆盖：

1. 默认展示数量为 `8`，可选值仅为 `2/4/8/12/16`，旧设置和非法值回退到 `8`。
2. 新设置只出现在自定义弹窗，不出现在首次偏好设置页面。
3. 常规计算严格保留设置数量的推荐方案。
4. 无冲突预览最多 `100` 条，同时返回准确总数；总数小于、等于和大于 `100` 均正确。
5. 点击“全部展示”返回全部无冲突方案；加载和失败状态不破坏已有结果。
6. 无冲突模式可返回推荐方案，并正确处理设置变化产生的新计算。
7. `1～9周 + 10～18周` 与 `1～18周` 的相同星期、节次、校区课程归入同一时间组。
8. 存在周次间隙或时间、校区不同的课程不会误合并。
9. 现有排课排序、冲突计算、设置弹窗样式和课程详情信息保持回归通过。

实现完成后运行单元测试、类型检查、代码检查和生产构建，并在浏览器中验证设置下拉框、无冲突方案切换、底部总数文案及时间组合并结果。
