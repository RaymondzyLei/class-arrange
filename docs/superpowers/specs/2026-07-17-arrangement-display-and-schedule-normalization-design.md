# 排课方案展示数量与时间组归并设计

## 目标

本次修改包含两个相互独立但共同影响排课结果展示的范围：

1. 允许用户在“自定义”设置中选择常规展示的排课方案数量，并能切换查看冲突最少的前 `100` 个方案。
2. 修正课程时间表分段方式不同导致同一时间组未被合并的问题。

不改变首次进入时的偏好设置，不修改现有开关样式，也不改变排课方案的排序规则。

## 自定义设置

在“自定义 → 课表生成”分组新增“展示排课方案数量”下拉框，选项固定为 `2`、`4`、`8`、`12`、`16`，默认值为 `8`。

该设置不出现在首次进入时的偏好设置页面。设置变更后触发重新排课，常规方案列表按用户选择的数量保留和展示；设置需要与其他自定义设置一样持久化。读取旧版本设置时，缺少该字段应自动使用默认值 `8`。

## 更多方案展示

排课方案板块增加独立按钮“展示更多方案”。它不是设置项，也不改变用户保存的常规展示数量。

每次排课仍精确遍历全部组合。所有候选方案不区分是否冲突，统一先按冲突数从少到多排序，再使用现有偏好和稳定键作为次级排序。计算引擎只维护排序最靠前的 `100` 个方案，并同时统计全部候选方案的实际数量。

常规视图显示 Top-100 结果中的前 N 个，N 为用户设置的展示数量。点击“展示更多方案”后显示 Top-100 全部结果；如果实际方案不足 `100` 个，则显示全部实际结果。扩展视图提供“返回推荐方案”，并保持常规视图的选择状态。

扩展视图底部信息规则如下：

- 总数不超过 `100`：显示“共 xx 种排课方案”。
- 总数超过 `100`：显示“共 xx 种排课方案，当前展示冲突最少的前 100 种。”
- 总数为 `0`：显示明确的无结果提示，并允许返回推荐方案。

界面不提供“全部展示”或无限加载按钮，也不会为扩展视图再次运行组合搜索。

## 计算接口与状态

计算引擎返回结构化结果：排序最靠前的最多 `100` 个方案，以及全部候选方案的精确总数。Worker 只传输这份有界结果，不传输完整笛卡尔积。

应用层从同一份 Top-100 结果派生常规视图和扩展视图。任何设置或课程选择变化都退出扩展视图并以新输入重新计算。

方案选择继续使用稳定的方案 ID。返回推荐模式时恢复此前选中的推荐方案；扩展视图中的选择只在该输入仍有效时保留。

## 时间组归并修复

时间组归并比较“实际占用时间”，而不是比较原始时间段数组如何切分。

生成时间指纹时，将每个日程展开为按周的原子占用单元。每个单元包含：

- 周次；
- 星期；
- 节次，或无节次数据时的精确起止时间；
- 校区。

教室继续按照现有规则不参与时间组指纹。展开后的单元去重、排序后生成规范指纹。因此，`1～9周 + 10～18周` 与 `1～18周` 在星期、节次和校区相同时得到相同指纹，能够归入同一时间组；如果中间缺周、星期不同、节次不同或校区不同，则仍属于不同时间组。

规范化只用于比较时间占用。课程原始教师、教室和分段日程继续保留，详情展示不虚构“某位老师对应某段时间”的映射，因为当前课程数据只提供合并后的教师字段，没有逐日程教师关系。

## 错误与边界处理

- 持久化设置值不在允许选项中时回退到 `8`。
- 方案总数以实际精确遍历结果为准，不使用估算值。
- 计算引擎最多保留和传输 `100` 个方案，不因实际总数增加而无限占用内存。
- 日程缺少周次时保持现有语义，不能因规范化而与具有明确周次的日程误合并。

## 测试与验收

自动化测试至少覆盖：

1. 默认展示数量为 `8`，可选值仅为 `2/4/8/12/16`，旧设置和非法值回退到 `8`。
2. 新设置只出现在自定义弹窗，不出现在首次偏好设置页面。
3. 常规计算严格保留设置数量的推荐方案。
4. 扩展结果最多 `100` 条，同时返回全部候选方案的准确总数；总数小于、等于和大于 `100` 均正确。
5. 推荐视图显示设置数量，扩展视图显示最多 `100` 条，且两者均按冲突数从少到多排序。
6. 扩展视图没有“全部展示”按钮，可返回推荐方案，并正确处理设置变化产生的新计算。
7. `1～9周 + 10～18周` 与 `1～18周` 的相同星期、节次、校区课程归入同一时间组。
8. 存在周次间隙或时间、校区不同的课程不会误合并。
9. 现有排课排序、冲突计算、设置弹窗样式和课程详情信息保持回归通过。

实现完成后运行单元测试、类型检查、代码检查和生产构建，并在浏览器中验证设置下拉框、更多方案切换、底部总数文案及时间组合并结果。
