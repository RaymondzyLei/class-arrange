# 方案分享链接设计

## 目标

让用户在没有账号和后端服务的情况下，把当前学期的一个选课方案作为快照分享给其他用户。接收方通过链接预览方案，并将仍然有效的课堂导入本地。

首版只分享当前方案的名称和已选课堂，不分享收藏、培养方案选择、自定义占位、排课倾向或其他本地设置。分享是一次性快照，不会随发送方后续修改而更新，也不支持共同编辑。

## 分享格式

分享数据使用带版本号的紧凑 JSON：

```json
{"v":1,"s":"2026-fall","n":"无早八","c":["MATH1006.01"]}
```

字段含义：

- `v`：分享格式版本，首版固定为 `1`。
- `s`：学期 key，必须存在于当前学期索引。
- `n`：来源方案名称，最长 20 个字符。
- `c`：已选课堂 ID，去重并保持原顺序，最多 100 个。

JSON 使用 UTF-8 编码后转换为 Base64URL，并写入 URL fragment：

```text
https://example.test/#plan=<base64url>
```

fragment 不会发送给网站服务器。链接不是加密载荷，任何获得链接的人都能读取其中的方案名称和课堂 ID；分享界面应提示用户不要把链接当作私密凭证。

最终生成的完整 URL 不得超过 1800 个字符。首版不增加压缩依赖；超过限制时拒绝生成并提示用户。

## 发送方交互

「我的方案」主操作行按以下顺序排列：

1. 方案选择框
2. 新建方案
3. 分享当前方案
4. 删除当前方案
5. 更多操作

分享按钮位于新建与删除之间。新增按钮占用的水平空间由方案选择框让出；选择框保留最小可用宽度、文本省略和完整名称提示，并继续适配移动端。

没有当前方案或当前方案为空时禁用分享按钮。点击分享按钮打开分享弹窗，弹窗展示方案名、所属学期、课程数量和只读链接，并提供：

- 「复制链接」作为通用主操作。
- 浏览器支持 `navigator.share()` 时提供「系统分享」。

首版不提供二维码。生成链接不会改变发送方当前页面的地址。

## 接收方数据流

应用启动或 fragment 变化时读取 `#plan`：

1. 解码 Base64URL 并验证 JSON 结构、版本和字段边界。
2. 若目标学期与当前学期不同，调用现有 `switchSemester()` 加载目标课程目录。
3. 目标学期加载完成后，使用该学期 `courseMap` 把课堂分为有效与失效两组。
4. 弹出只读预览，显示方案名、学期、有效课程列表和失效课堂 ID。
5. 用户确认后，通过一条原子 reducer action 导入并激活方案。
6. 用户导入、取消或关闭错误提示后，使用 `history.replaceState()` 清除分享 fragment，避免刷新时重复处理。

取消导入后页面继续停留在分享方案所属学期，不恢复打开链接前的学期。

## 导入规则

若本地当前学期只有一个方案且该方案 `courseIds.length === 0`，则复用这个空方案；方案名称、历史是否选过课以及是否曾被重命名都不参与判断。

其他情况一律创建新方案，不覆盖或合并现有方案。若当前已经有 10 个方案，仍允许预览，但禁用导入并提示用户先删除一个方案。

有效课堂部分导入，失效课堂不导入并在预览中警告。若全部课堂均失效，则禁止导入。

导入名称按以下顺序计算：

1. 来源名称符合默认方案模式（例如「方案一」「方案二」）时，不保留来源序号，而是使用接收方下一个可用默认方案名。
2. 自定义名称未重复时原样保留。
3. 自定义名称重复时依次使用「名称 副本」「名称 副本 2」。
4. 复用唯一空方案时，名称计算忽略这个即将被替换的方案。

导入 action 同时完成方案复用或创建、课程写入、时间戳更新和 active plan 切换，避免出现中间状态。

## 组件与模块

### `src/utils/sharedPlan.ts`

提供无 UI 依赖的纯函数：

- 分享载荷类型和验证结果类型。
- UTF-8 Base64URL 编码与解码。
- `encodeSharedPlan()` / `decodeSharedPlan()`。
- 分享 fragment 与完整 URL 的生成。
- 导入名称计算。
- fragment 清除。

### `src/hooks/useSharedPlanImport.ts`

负责：

- 监听并解析当前 fragment。
- 驱动跨学期切换。
- 在目录加载完成后匹配有效与失效课堂。
- 管理预览、错误、导入禁用原因和关闭行为。

该 hook 不直接渲染界面，也不自行写入 `localStorage`。

### `src/components/SharePlanModal.tsx`

发送方弹窗，只接收当前方案、学期信息和开关状态，负责生成、展示、复制或调用系统分享。

### `src/components/SharedPlanImportModal.tsx`

接收方预览弹窗，展示有效课程、失效课堂和导入限制，通过回调请求确认或取消。

### `src/store/plansReducer.ts`

增加原子导入 action。复用空方案和新建方案的状态变更由 reducer 统一执行。

### `src/components/PlanSwitcher.tsx` 与 `src/App.tsx`

`PlanSwitcher` 增加分享按钮和发送弹窗入口。`App` 连接学期目录、方案状态、导入 hook 与接收方预览弹窗。

## 错误处理

以下情况不得修改本地方案：

- Base64URL 无法解码。
- JSON 无法解析。
- 分享格式版本不支持。
- 字段缺失或类型不正确。
- 方案名、课堂数量、课堂 ID 或载荷长度超过限制。
- 目标学期不在当前 manifest 中。
- 课程目录切换失败。
- 所有分享课堂均已失效。
- 本地已有 10 个方案且没有可复用的唯一空方案。

无效或损坏的分享链接显示明确错误弹窗。关闭后清除 fragment。课程部分失效不视为整体错误，而是在预览中警告并允许导入其余课程。

剪贴板写入或系统分享失败时保留弹窗和链接文本，提示用户重试或手动复制。

## 测试与验收

### 单元测试

- 中文方案名和课堂 ID 的编码解码往返。
- Base64URL 只包含 URL-safe 字符。
- 课堂 ID 去重并保持顺序。
- 截断载荷、错误 JSON、未知版本、字段缺失和边界超限。
- 默认名、自定义名、重复副本与复用空方案的命名规则。
- reducer 对唯一空方案的复用和对已有方案的新建、激活行为。

### 流程测试

- 同学期分享预览与导入。
- 跨学期自动切换后预览。
- 部分课堂失效仍可导入。
- 全部课堂失效时禁止导入。
- 10 个方案时禁止新增导入。
- 导入、取消与错误关闭后清除 fragment。

### UI 与回归验收

- 操作按钮顺序为新建、分享、删除、更多。
- 空方案的分享按钮禁用。
- 桌面端和移动端方案选择框保持可用，长名称正确省略。
- 分享与导入弹窗在明暗主题下可读。
- 完整 Vitest、oxlint 和生产构建通过。
- 在真实浏览器中验证复制链接、链接打开、跨学期导入和响应式布局。
